🤖 AI Tool for Generating Tamarin Rules

From Protocol Diagrams to Formal Verification Models

Task 1 Submission - Formal Verification of Security Protocols

📋 Overview

🎯 Task 1 Objective

Given a protocol diagram (sequence of rounds), develop an AI tool that generates the Tamarin rules for the protocol.

Instructor: Prof. Raghavendra Singh Rohit, IIT Roorkee

This tool automatically converts protocol sequence diagrams (PNG, JPG, PDF) into formal Tamarin verification models (.spthy files) using a hybrid approach combining Computer Vision (OCR), Large Language Models (GPT-4), and Template-Based Code Generation.

🏗️ AI Pipeline Architecture

1

📷 Input: Protocol Diagram

Accepts protocol diagrams in PNG, JPG, or PDF format showing message sequences between participants (Alice, Bob, Server, etc.)

↓
2

🧠 AI Component 1: Vision/OCR

Tesseract OCR + OpenCV extracts text, detects roles, identifies message arrows (→), and parses cryptographic operations (enc, dec, sign, verify)

↓
3

🤖 AI Component 2: LLM Enhancement

GPT-4 infers protocol semantics, detects fresh values (nonces, keys), and suggests security properties (authentication, secrecy lemmas)

↓
4

📝 Template Generator

Applies Tamarin patterns from DigiLocker slides to generate syntactically correct rules with proper equational theories and action facts

↓
5

✅ Output: Tamarin Model

Generates complete .spthy file ready for formal verification with Tamarin prover, including all rules and security lemmas

📊 Example: DigiLocker Aadhar Authentication

Input (Protocol Flow)

User → DigiLocker: Aadhaar + OTP DigiLocker → UIDAI: sign(aenc(AuthXML, pk_UIDAI), ltk_DigiLocker) UIDAI → DigiLocker: sign(aenc(Response, pk_DigiLocker), ltk_UIDAI) DigiLocker → User: Authenticated

Output (Generated Tamarin Rules)

theory DigiLocker_Aadhar_Authentication begin builtins: hashing, asymmetric-encryption, signing // PKI Setup rule RegisterKeys: [ Fr(~ltkD), Fr(~ltkA) ] --> [ !Ltk($DigiLocker, ~ltkD), !Ltk($UIDAI, ~ltkA), !Pk($DigiLocker, pk(~ltkD)), !Pk($UIDAI, pk(~ltkA)) ] // Protocol Execution rule DigiLocker_Send_Step2: let msg = sign(aenc(<~N, <~O, ~Tx>>, pkA), ltkD) in [ Fr(~N), Fr(~O), Fr(~Tx), !Ltk($DigiLocker, ltkD), !Pk($UIDAI, pkA) ] --[ SentAuth($DigiLocker, $UIDAI, ~N, ~O, ~Tx) ]-> [ Out(msg) ] // Security Properties lemma authentication: "All D A tx #i. TrustEstablished(D, tx) @ i ==> Ex #j. UIDAIResponded(A, D, tx) @ j & j < i" end
